12. Custom panel detection

Implement an intelligent detector that decides whether a selected element or section is a real-control container or custom-drawn panel.

Signals of custom-drawn panel:

* Win32 child count is 0 or very low
* MSAA child count is 0
* UIA Raw View returns only Pane/Client/Custom
* Multiple click points inside panel return same HWND
* Element names are empty
* OCR finds multiple text blocks inside the panel
* Screenshot shows multiple button-like regions
* Control class is PictureBox/UserControl/custom canvas-like
* No Invoke/Value/Text/Grid patterns are available

Implement scoring:

If score >= threshold, classify as CustomDrawnPanel.

Example pseudo-code:

public bool IsCustomDrawnPanel(InspectionResult result)
{
int score = 0;

```
if (result.Win32ChildCount == 0) score++;
if (result.MsaaChildCount == 0) score++;
if (result.UiaChildCount <= 1) score++;
if (result.OcrTextBlocks.Count >= 3) score++;
if (result.DetectedButtonRegions.Count >= 3) score++;
if (result.SameHwndForMultiplePoints) score++;
if (result.PrimaryClassName.Contains("PictureBox") || result.PrimaryClassName.Contains("UserControl")) score++;

return score >= 3;
```

}

public ActionResult Click(DesktopElement element, ActionValidation validation)
{
foreach (var locator in element.Locators.OrderByDescending(l => l.Score))
{
try
{

```
        if (resolved == null)
            continue;

        var result = actionExecutor.Click(resolved);

        if (!result.Success)
            continue;
```

30. Critical design principles

* Do not depend on only UIA.
* Do not depend on only OCR.
* Do not depend on absolute coordinates.
* Always collect evidence from multiple layers.
* Always generate multiple locators.
* Always rank locators by stability and confidence.
* Always validate after action.
* Always retry with fallback.
* Always capture diagnostics when failure happens.
* Reconstruct virtual elements when real child elements do not exist.
* Treat POS automation as workflow automation, not just element automation.
* Prefer backend/log validation where available.
* Use coordinate fallback only as the last option.
* Make the inspector explain why a locator was selected.
